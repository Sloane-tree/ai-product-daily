## 🔥 Today's Signals

### 1. Karpathy 泼了盆冷水：逼 Agent 干活是 AI 最大错误

**Insight**：Karpathy 在 Anthropic 内部分享了一个反直觉的观点——当前 AI 领域最大的错误，是人们急着让 Agent 去干活，却没先把底层模型搞明白。他搬出了自己 2016 年在 OpenAI 做的 World of Bits 项目——让 Agent 订机票、点外卖——和今天 Agent 创业公司的 PPT 第一页几乎一模一样。当时失败了，因为手里只有强化学习这把锤子。回头看，正确的做法是"彻底忘掉 Agent，转头做语言模型"。

**落地过程**：Karpathy 今年 5 月加入 Anthropic 预训练团队，这本身就是一记"行为投票"。他发明了 vibe coding 这个词，却选择回到最底层做模型。他从神经科学找灵感——Agent 里的"海马体"（记忆检索）、"基底神经节"（行为选择）、"丘脑"（多念头抢麦克风）——而不是从工程架构出发。同时他明确说 vibe coding 适合原型探索，但专业开发正进入 Agentic Engineering 阶段——需要更多监督和可验证交付物。

**对我的启发**：Karpathy 的核心信息不是"别做 Agent"，而是"别跳过基础做 Agent"。这对产品经理意味着——选 Agent 平台时，先评估底层模型能力，而不是看 Agent 框架的花哨功能。另外他说的"Agent 不是产品，基础能力才是产品"值得贴在墙上——你做的是产品，不是套壳。

### 2. Replit CEO：让 Agent 自己读用户日志、找到 bug、提 PR、自己修

**Insight**：Amjad Masad 在 SaaStr AI 2026 上分享了一个细节——Replit 内部跑了一个 Agent，每天晚上自动读取所有用户的使用 trace，发现 bug 后自己生成 prompt 修复方案、提 PR、跑 A/B test。全程无人干预。这不是 demo，是已经跑在生产环境里的东西。

**落地过程**：他们发现关键在于"monorepo 优势"——多个 App 跑在同一个代码库里，Agent 能拿到全局上下文，效果远好于单项目孤立运行。经济学上也成立：这套 Agent 系统每月成本 $257，替代的是年薪 $140K+ 的工程师。另外他提到 Replit 的 AI 写 B2B 外发邮件已经超过大多数人类——不是"未来"，是"现在"。

**对我的启发**：自愈型 Agent 不是科幻。Replit 这个案例给了产品经理一个可参考的架构——不是从头训练模型，而是"Agent + 日志数据 + 修复流水线"。如果你的产品有用户行为数据，就可以考虑类似的闭环——Agent 观察用户怎么用的 → 发现哪里卡住了 → 自动优化。

### 3. Cursor 设计负责人 Ryo Lu：画按钮的时代，真的结束了

**Insight**：Cursor 设计主管 Ryo Lu 提出了一个激进的观点——设计师的工作不再是画界面，而是定义"Agent 的行为边界"。他加入 Cursor 时公司没有一个全职设计师，但他做的第一件事不是画 UI，而是把 Cursor 五个分散的功能（Tab 补全、Command K、Chat、Composer、Agent）统一成一个 Agent 概念。据称这是 Cursor 2025 年高速增长的核心原因。

**落地过程**：Ryo 的 Design Mode 颠覆了传统设计工具的逻辑——不再是拖拽组件，而是你在浏览器里点击元素、画标注、说一句话，Agent 就帮你改好了。他自己用 Cursor 一个人写了 13 万行代码的 RyOS（基于 Web 的 Agent 操作系统）。他的核心方法论是"把 AI 当联合创始人，不是员工"——不写详细 spec，而是描述问题，让 Agent 一起碰撞方案。

**对我的启发**：Ryo 做的"统一 Agent 模式"本质上是一个产品架构决策，不是设计决策。他把五个功能合成一个，因为用户心智里不需要五个 AI 功能——只需要"一个能帮我干活的东西"。这对产品经理的启发：AI 产品的功能不是越多越好，把分散的 AI 能力收拢成一个统一的 Agent 入口，可能比加十个新功能更有价值。

### 4. GitHub 本周最火项目不是模型，是"让 AI 说话更省钱"

**Insight**：本周 GitHub Trending 第一不是新模型，而是一个叫 caveman 的项目（85K ⭐）——让 Claude 把输出压缩成穴居人风格，省 65% token。配合 superpowers（245K ⭐，Agent Skills 框架）和 agentskills（22K ⭐，Skills 标准化规范），三个项目指向同一个趋势：Skills 正在变成 Agent 时代的 App Store。

**落地过程**：caveman 只做一件事——把 Agent 内部通信的语言精简到极致，但用户可见的输出不变。背后是行业的集体焦虑：token 成本正在成为瓶颈。OpenAI 砍了推理价格、DeepSeek 推出峰谷计费、caveman 从语言层面压缩——三件事同时发生，说明"成本工程"已经是一个独立的产品方向。

**对我的启发**：对产品经理来说，这意味着两件事。一是设计 AI 产品时要考虑 token 成本——你的 prompt 设计直接决定了每次调用的成本。二是 Skills 正在成为分发单位——就像 iOS 的 App Store 让开发者不需要懂芯片就能做 app，Skills 让领域专家不需要懂 prompt engineering 就能封装自己的知识。你应该开始积累自己的 Skills，而不是手写 prompt。

### 5. OpenAI 给 Claude Code 写了个插件——生态围墙正式倒塌

**Insight**：本周最意味深长的 GitHub 项目不是技术最强的那一个，而是 OpenAI 官方的 codex-plugin-cc（24K ⭐）——让 Claude Code 用户能在终端里直接调用 OpenAI Codex。OpenAI 亲手为竞品写插件。

**落地过程**：这背后的逻辑很简单——开发者已经住在 Claude Code 里了，OpenAI 的选择是"加入"而不是"对抗"。这验证了一个趋势：模型能力不再是护城河，用户习惯才是。谁能占领开发者的终端，谁就有分发权。OpenAI 宁愿在 Claude Code 里当"被调用的服务"，也不愿意赌开发者会换工具。

**对我的启发**：如果你的产品依赖某个 AI 平台，现在就要做"模型抽象层"。不是过度工程——OpenAI 自己都在拥抱竞品生态。同时这也是一个产品策略参考：如果你的竞品已经占领了用户心智（比如 Claude Code 是"靠谱老员工"），不要正面进攻，学 OpenAI 的方式——把你自己嵌入到竞品的工作流里。

### 6. Product Hunt 上出现了一个有意思的产品：Glaze——描述一个 app，给你一个真的桌面应用

**Insight**：Raycast 团队做的 Glaze 不是"AI 帮你写代码"，而是"你用自然语言描述想要什么 app，AI 直接在 macOS 上给你生成一个可以离线运行的真实桌面应用"。这比传统的 AI coding 工更进一步——用户连"看代码"这一步都跳过了。

**落地过程**：Glaze 选择了 macOS 原生路线，生成的 app 不是网页套壳，是真的能进 Dock、能离线用。这和 Raycast 一贯的产品哲学一致——"让用户留在桌面，不要打开浏览器"。技术上的关键是限制了生成范围（只做桌面小工具），反而让体验非常完整。

**对我的启发**："用自然语言生成一个 app"这个赛道正在从"能做"走向"好用"。Glaze 的关键选择是范围控制——不是做一个万能的 app 生成器，而是只做桌面小工具这个窄场景。这正是 Zara Zhang 说的"从小而怪的产品出发"——窄定位反而更容易做到极致。

## 💡 Today's Builder Insight

**Karpathy 和 Replit 在同一天讲了同一件事，但从相反的方向。**

Karpathy 说"别急着做 Agent，先把底层模型搞对"。Replit 说"Agent 已经在生产环境自愈了，每月成本 $257"。表面上是矛盾的，但底层逻辑一致——两者都在说"认真对待 Agent 的基础设施"。

Karpathy 的"基础"是模型层——他回到 Anthropic 预训练团队，从神经元级别做起。Replit 的"基础"是工程层——monorepo 让 Agent 有全局上下文、自动备份让 Agent 出错可以回滚、沙箱让 Agent 不会删生产数据库。

这个矛盾恰好定义了 2026 年 AI 产品经理的核心张力——你的产品是应该在"等模型变好"还是"现在就用工程手段解决问题"？答案可能是两者都要。**模型没到那一步的时候，用工程补；模型到了的时候，工程积累就是护城河。**
